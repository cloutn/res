采用表格编辑器：
优点：
1.隔绝策划和表格，使得程序在表格的设计上有更大的灵活性和自主权。
只要给策划的是一个他要求的逻辑试图，背后的存储逻辑程序完全可以根据自己需要修改。

2.在完全不影响数据结构定义、界面、策划的情况下，程序可以随意选择序列化的方式（txt，二进制，xml）,甚至可以实现一份视图和数据可以同时导出为不同文件格式。
序列化的功能主要通过可定制的ISerializable接口来实现，目前实现了txt和二进制，今后可以增加XML格式。

3.在编辑器中增加对逻辑的检查，减少由于策划配置疏忽导致的程序帮助策划查问题的时间开销。
例如使用传送点编辑器之后，对于很多细节，编辑器来保证，如果策划有问题编辑器会先提示，而不是等到程序来调试才发现问题。

4.降低策划对表格细节的依赖，使其可以专心于策划逻辑的设定，而无需反复思考表格之间的逻辑父子关系、外键关系等。
从这个角度讲，编辑器可以降低策划配置的门槛。
其实编辑器也方便程序快速制作一些自己需要的临时测试数据，即便这些功能模块和表格逻辑自己并不熟悉，因此可以说也间接提升了程序的开发效率。

5.可以针对游戏逻辑制作一些易用的界面，例如金钱编辑Form，概率编辑Form等等，提高策划效率。

6.编辑器采用了界面层，数据层，序列化层的分层结构，有较高的可复用性。
目前由于是demo，界面层部分分离不完善，但是短期内可以解决。



缺点：
1.编辑器在灵活性方面短期内无法和excel抗衡。
易用性方面可以根据策划需要随时修改，可以尝试实现某些策划常用的宏或脚本，但是对于习惯于excel宏和脚本的人来说，编辑器很难达到那个高度，毕竟不是专业的表格制作软件。

2.编辑器需要程序人员维护开发，有较高的开发成本。
该开发成本在游戏完整性提高后会急剧下降，更多是细节完善。

3.编辑器应对需求的快速反应能力肯定不如直接改excel来的快。
但是使用编辑器并不代表不能用excel（在使用目前txt格式的情况下），因此在某些特殊情况下完全可以依然直接用excel改表格。

4.目前编辑器很多基本细节需要完善，但是都是时间问题，比如批量处理问题，在短期内都可以解决。

5.目前程序中有130多张表格，这些表格要纳入编辑器需要一定的时间，时间主要会消耗在是表格之间逻辑的整理上。
对于该问题，我们可以采用渐进式的方式，逐步将一些表格纳入到编辑器内。

重装系统后如何恢复svn服务器？
1.安装好apache和svn
2.把备份的apache的配置文件拷贝进去，svn的mod文件拷贝到apache下面
3.重启电脑
4.重启apache服务
5.注意windows防火墙的80端口打开
6.注意本地ip是192.168.1.2（即和路由器配置一样，路由器配置是192.168.1.2:80）。
7.局域网内另外一台电脑如果可以访问svn，基本就没啥问题了。

vs2008编译错误fatal error C1902: 程序数据库管理器不匹配；请检查安装解决 收藏
 重装了本本上的Xp系统，如往常一样，升级，装杀毒软件，开发工具。一些进行的非常顺利。然而，在我打开VS2008准备耕作的时候，尽然出现了一邪恶的错误提示：vs2008编译错误fatal error C1902: 程序数据库管理器不匹配；请检查安装解决。进过多家的方法，均无效，重装VS也不行，最后终于百度到一无名大侠的解决方案，问题解决了，在这里记录下来，以备再有不测。
到C:\Program Files\Microsoft Visual Studio 9.0\Common7\IDE下面拷贝mspdbsrv.exe到C:\Program Files\Microsoft Visual Studio 9.0\VC\bin下面，重启VS即可。
问题的原因我还不清楚，也学是DLL版本混乱的问题...
